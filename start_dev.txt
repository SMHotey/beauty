ВАРИАНТ 1: Запуск без Docker (рекомендуется для разработки)
============================================================

ШАГ 1. Настройка бэкенда
-------------------------

1.1. Откройте терминал и перейдите в директорию проекта:
    cd "C:\Users\malik\OneDrive\Рабочий стол\Projects\Beauty"

1.2. Перейдите в папку бэкенда:
    cd backend

1.3. Файл .env уже существует. При необходимости скопируйте из примера:
    copy .env.example .env

1.4. Активируйте виртуальное окружение (уже создано в корне проекта):
    ..\venv\Scripts\activate

    Если venv нет — создайте:
    python -m venv ..\venv
    ..\venv\Scripts\activate

1.5. Установите зависимости:
    pip install -r requirements\dev.txt

1.6. Примените миграции базы данных:
    python manage.py migrate

1.7. Загрузите начальные данные (услуги, мастера, настройки):
    python manage.py loaddata fixtures/initial_data.json

1.8. Создайте суперпользователя (администратора):
    python manage.py createsuperuser
    Следуйте инструкциям в терминале.
    Или используйте готовые данные (если есть в фикстурах):
    Email: admin@example.com
    Пароль: admin123

1.9. Запустите Django-сервер:
    python manage.py runserver
    Бэкенд будет доступен по адресу: http://localhost:8000


ШАГ 2. Запуск Celery (опционально, для напоминаний)
----------------------------------------------------

Откройте НОВЫЙ терминал, перейдите в backend и активируйте venv:

2.1. Celery worker:
    cd backend
    ..\venv\Scripts\activate
    celery -A beauty_backend worker -l info

2.2. Celery beat (планировщик задач):
    cd backend
    ..\venv\Scripts\activate
    celery -A beauty_backend beat -l info

    Примечание: на Windows Celery требует пакет eventlet:
    pip install eventlet
    celery -A beauty_backend worker -l info -P eventlet


ШАГ 3. Настройка и запуск фронтенда
-------------------------------------

Откройте НОВЫЙ терминал:

3.1. Перейдите в папку фронтенда:
    cd "C:\Users\malik\OneDrive\Рабочий стол\Projects\Beauty\frontend"

3.2. Установите зависимости (если ещё не установлены):
    npm install

3.3. Создайте файл .env из примера (если его нет):

3.4. Запустите dev-сервер:

    Фронтенд будет доступен по адресу: http://localhost:5173

ВАРИАНТ 2: Запуск через Docker
===============================

ШАГ 1. Перейдите в корень проекта:
    cd "C:\Users\malik\OneDrive\Рабочий стол\Projects\Beauty"

ШАГ 2. Убедитесь, что файл backend/.env существует:
    copy backend\.env.example backend\.env

ШАГ 3. Запустите все сервисы одной командой:
    docker-compose up --build

    Это запустит:
    - PostgreSQL (порт 5432)
    - Redis (порт 6379)
    - Django backend (порт 8000)
    - Celery worker (напоминания)
    - Celery beat (расписание задач)
    - React frontend (порт 5173)
    - Nginx (порт 80)

ШАГ 4. Откройте приложение:
    - Фронтенд: http://localhost:5173
    - API Swagger: http://localhost:8000/api/docs/swagger/
    - Django Admin: http://localhost:8000/admin/

ШАГ 5. Остановка сервисов:
    docker-compose down

    Для удаления всех данных (включая БД):
    docker-compose down -v


ПОЛЕЗНЫЕ КОМАНДЫ
=================

Бэкенд:
  python manage.py makemigrations     — создание новых миграций
  python manage.py migrate            — применение миграций
  python manage.py createsuperuser    — создание администратора
  python manage.py loaddata fixtures/initial_data.json — загрузка данных
  pytest                              — запуск тестов
  pytest --cov=apps                   — тесты с покрытием

Фронтенд:
  npm run dev                         — dev-сервер
  npm run build                       — сборка для продакшена
  npm run preview                     — предпросмотр собранной версии
  npm run lint                        — проверка кода линтером

Docker:
  docker-compose up --build           — запуск всех сервисов
  docker-compose down                 — остановка сервисов
  docker-compose down -v              — остановка + удаление данных
  docker-compose logs -f backend      — логи бэкенда
  docker-compose logs -f frontend     — логи фронтенда

УСТРАНЕНИЕ НЕПОЛАДОК
=====================

Проблема: Ошибка подключения к БД
Решение: Убедитесь, что в backend/.env указан правильный DATABASE_URL
         Для SQLite (по умолчанию): DB_ENGINE=django.db.backends.sqlite3, DB_NAME=db.sqlite3
         Для PostgreSQL: DATABASE_URL=postgres://beauty:beauty123@localhost:5432/beauty

Проблема: Ошибка CORS при запросах с фронтенда
Решение: Убедитесь, что в backend/.env есть:
         CORS_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000

Проблема: Celery не запускается на Windows
Решение: Установите eventlet и используйте флаг -P:
         pip install eventlet
         celery -A beauty_backend worker -l info -P eventlet

Проблема: Порт уже занят
Решение: Найдите и завершите процесс:
         netstat -ano | findstr :8000
         taskkill /PID <PID> /F
